# Config Editor - Веб-приложение для редактирования конфигурационных файлов

**Config Editor** — это мощное и безопасное веб-приложение для системных администраторов, написанное на Python (Flask, Gunicorn), которое позволяет редактировать конфигурационные файлы на Linux-сервере через удобный веб-интерфейс.

Приложение спроектировано с учетом максимальной безопасности и гибкости. Оно использует системную аутентификацию пользователей и выполняет все файловые операции с привилегиями вошедшего пользователя, уважая права доступа, установленные в операционной системе.

## Основные возможности

- **Современный UI**: Чистый и отзывчивый интерфейс с деревом файлов, областью для редактирования и переключением тем (светлая/тёмная).
- **Множество форматов**: Встроенная поддержка `ini`, `conf`, `cfg`, `json`, `yaml` и других форматов. Система парсеров является модульной и легко расширяемой.
- **Безопасность превыше всего**:
    - **Системная аутентификация**: Используются логины и пароли системных пользователей Linux через PAM.
    - **Олицетворение привилегий**: Все операции с файлами (чтение/запись) выполняются от имени вошедшего пользователя, а не от пользователя веб-сервера.
    - **Ограниченный `sudo`**: Сервис работает от имени низкопривилегированного пользователя (`config-editor`), которому разрешено выполнять через `sudo` только один строго определенный скрипт-помощник.
- **Резервное копирование**: Встроенный механизм создания и восстановления зашифрованных (AES-256-GCM) резервных копий.
- **Гибкая настройка**: Все ключевые параметры (порт, пути, фильтры IP) настраиваются в едином файле `config.yaml`.
- **Простая установка**: Весь процесс развертывания автоматизирован с помощью одного скрипта `install.sh`.
- **Работа в качестве сервиса**: Приложение устанавливается как `systemd`-сервис, что обеспечивает его автоматический запуск при старте системы.

## Архитектура

- **Бэкенд**: Python 3, Flask, Gunicorn (в качестве WSGI-сервера).
- **Фронтенд**: Ванильный JavaScript (ES6+), HTML5, CSS3.
- **Аутентификация**: `python-pam`.
- **Парсеры**: Модульная система с обработчиками для разных форматов.
- **Шифрование**: `cryptography` (AES-256-GCM, PBKDF2).

## Установка

Процесс установки полностью автоматизирован. Вам нужно лишь выполнить один скрипт от имени `root`.

1.  **Клонируйте репозиторий**:
    ```bash
    git clone <URL вашего репозитория>
    cd config-editor
    ```

2.  **Запустите скрипт установки**:
    Скрипт должен быть запущен с правами суперпользователя.
    ```bash
    sudo ./install.sh
    ```

Скрипт выполнит все необходимые действия:
- Установит системные зависимости (`python3`, `pip`, `venv`, `libpam-dev`).
- Создаст системного пользователя `config-editor`.
- Скопирует приложение в `/opt/config-editor`.
- Настроит виртуальное окружение и установит Python-пакеты.
- Сконфигурирует `sudo` для безопасного выполнения файловых операций.
- Создаст и запустит `systemd`-сервис `config-editor.service`.

## Первоначальная настройка

После успешной установки приложение будет запущено, но потребует первоначальной настройки.

1.  **Доступ к приложению**:
    По умолчанию сервис использует UNIX-сокет (`/opt/config-editor/gunicorn.sock`). Для доступа к нему через браузер рекомендуется настроить **reverse proxy**, например, с помощью Nginx. Это также позволит легко добавить HTTPS.

    Если вы хотите быстро протестировать приложение без reverse proxy, вы можете временно изменить файл сервиса (`/etc/systemd/system/config-editor.service`), заменив `unix:/opt/config-editor/gunicorn.sock` на `0.0.0.0:5000` в строке `ExecStart`, а затем выполнить `sudo systemctl daemon-reload` и `sudo systemctl restart config-editor`. **Не забудьте вернуть изменения обратно для безопасной работы!**

2.  **Настройка в браузере**:
    - Откройте приложение в браузере.
    - Вы будете перенаправлены на страницу `/setup`.
    - Установите пароль для системного пользователя `admin` (этот пароль используется только для управления бэкапами, для входа используются системные учетные данные).
    - Укажите **абсолютный путь** к директории, конфигурационные файлы в которой вы хотите редактировать (например, `/etc/` или `/home/user/configs`).
    - Нажмите "Complete Setup".

## Использование

- После первоначальной настройки вас перенаправит на страницу входа.
- Введите **логин и пароль любого системного пользователя Linux**, который имеет права на чтение/запись нужных вам конфигурационных файлов.
- После входа в левой панели появится дерево файлов из указанной вами директории.
- Права доступа (чтение/запись) будут отражены в интерфейсе.
- Выберите файл для редактирования.

## Управление

- **Логи**: `/var/log/config-editor/`
- **Файлы приложения**: `/opt/config-editor/`
- **Сервис**: `sudo systemctl status/start/stop/restart config-editor`
- **Конфигурация**: `/opt/config-editor/data/config.yaml` (требуются права `root` для редактирования после установки).
